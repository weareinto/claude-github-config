## Claude Code skills

Invoke with `/skill-name` in Claude Code.

| Skill | Purpose |
|---|---|
| `/start` | Pre-implementation gate: verify ticket, check Ready status, create branch, move to In progress |
| `/branch` | Create a correctly named branch linked to a GitHub issue |
| `/issue` | Create a GitHub issue from a template |
| `/pr-description` | Generate a PR description from the diff |
| `/pr-submit` | Submit a PR (description + changelog fragment + labels) |
| `/review` | Self-review checklist before opening a PR |
| `/changelog` | Create a changelog fragment for the current branch |
| `/claude-code-docs` | Reference for Claude Code features (hooks, skills, MCP, subagents, etc.) |

## Claude Code hooks

Hooks fire automatically before every file operation — no manual step needed.

| Hook | Triggers on | What it does |
|---|---|---|
| `protect-env.sh` | Read, Edit, Write | Blocks access to `.env` files (`.env.example` is allowed) |
| `protect-main.sh` | Edit, Write | Blocks file modifications when on `main` branch |

## GitHub workflows

| Workflow | Trigger | What it does |
|---|---|---|
| `auto-label.yml` | Issue opened | Applies `type:*` label from the issue template |
| `checklist-to-ready.yml` | Issue edited | Moves card Backlog → Ready when all checklist items are ticked |
| `inject-design-section.yml` | `needs:design` label added | Injects a `## Design/Mockup` section into the issue body |
| `assign-pr-to-project.yml` | PR opened/reopened | Adds the PR to the project board → In review |
| `request-copilot-review.yml` | PR opened/reopened | Requests Copilot as reviewer automatically |
| `generate-changelog.yml` | Manual | Compiles `CHANGELOG.md` from fragments, bumps version, opens release PR |
| `tag-on-release-merge.yml` | Release PR merged | Creates annotated git tag + GitHub Release |
| `sync-deploy-status.yml` | Push to `staging`/`production` | Moves linked issue cards → Staging / Production |
| `update-config.yml` | Manual | Pulls the latest claude-github-config template and opens a PR |

## After installation

| Step | Action |
|---|---|
| 1 | **Fill in `doc/PROJECT.md`** — Claude Code loads this at every session. Without it the AI has no project context. |
| 2 | Add the `CONFIG_PAT` secret (read access to `weareinto/claude-github-config`): `gh secret set CONFIG_PAT --repo weareinto/claude-github-config` |
| 3 | Verify the project board ([#22](https://github.com/orgs/weareinto/projects/22)) has all 9 Status columns — the installer creates any that are missing. |
| 4 | Fill in the tech stack section in `CONTRIBUTING.md` (or re-run the installer interactively). |
| 5 | Create `CLAUDE.local.md` (gitignored) with your personal Claude Code preferences. |
| 6 | Commit the installed files: `git add . && git commit -m "chore: apply claude-github-config"` |
